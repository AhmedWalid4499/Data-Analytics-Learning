Orientation

The one idea behind every flow

A flow is a trigger followed by one or more actions. The trigger is the "when"; the actions are the "then". Everything else — conditions, loops, variables — just makes the "then" smarter.

TRIGGER When a new email arrives with "GOLD order" in the subject
ACTION Post a message to the Delivery Teams channel
ACTION Add a row to the tracking spreadsheet
The four flow types

Pick the right starting point

⚡ Automated cloud flow

Starts on its own when an event happens — an email arrives, a list item changes, a file lands in SharePoint. The most common type.

Trigger example: When a new email arrives (V3)

👆 Instant cloud flow

You start it on demand — a button in the app, or from a Teams/Power Apps action. Can ask you for inputs first.

Trigger: Manually trigger a flow

⏰ Scheduled cloud flow

Runs on a clock — every Monday 8am, every hour, the 1st of the month. Built on the Recurrence trigger.

Trigger: Recurrence

🖥 Desktop flow (RPA)

Automates the screen — clicking buttons, typing into fields, reading a legacy web portal that has no connector. Runs in Power Automate Desktop.

For apps with no API — e.g. driving GOLD in a browser

i
Cloud vs Desktop: if there's a connector or API, stay in the cloud — it's more reliable. Reach for a desktop flow only when you must mimic a human clicking through an interface.
The building blocks

Every element, explained

These are the pieces you drag together inside a flow, grouped into three levels — tap a card to switch. Learn what each is for and you can build almost anything. Remember the spine: a trigger (the "when") followed by actions (the "then").

▸Trigger The "when" Core

The event that starts the flow. Exactly one per flow. Choosing it well matters: When an item is created vs created or modified changes everything downstream. Many triggers have filters (e.g. only emails with a certain subject) — use them to avoid firing on everything.

▸Action The "then" Core

A single step the flow performs: send an email, create a file, post to Teams, add a spreadsheet row, call an API. Actions run top to bottom; each can use outputs from the steps above it.

▸Connector The app it talks to Core

A bundle of triggers and actions for one service — Outlook, Teams, SharePoint, Excel Online, Planner, Approvals, Dataverse. Hundreds exist. Premium connectors (e.g. HTTP, SQL, custom) need the right licence.

▸Dynamic content Passing data between steps Core

The output of one step, dropped into a later one — the email's subject, the new item's ID, the file name. This is the wiring that makes a flow more than a list of unrelated actions.

i
If the dynamic value you need isn't offered, it's usually because the step that produces it is below the one you're editing, or inside a different branch.
▸Condition If / else Logic

Splits the flow into a Yes branch and a No branch based on a test (Priority is equal to "High"). Put the steps for each outcome in the matching branch.

Condition: Priority is equal to High ├─ If yes → send the manager an alert └─ If no → just add a log row
▸Switch Many branches at once Logic

Like a Condition but for several known values — branch on Status = Delivered / In Progress / On Hold, each with its own steps, plus a default. Cleaner than nesting many Conditions.

▸Apply to each Loop over a list Logic

When a step returns multiple items (rows from Excel, emails, list items), Apply to each runs its inner actions once per item. Power Automate often adds it for you the moment you use an array output.

!
Loops can be slow over big lists. Filter the data before the loop, and turn on concurrency in the loop's settings when order doesn't matter.
▸Do until Loop until true Logic

Repeats until a condition is met (e.g. keep polling a status until it reads "Complete"). Always set a sensible count/timeout limit so it can't run forever.

▸Variables Remember a value Logic

Initialise a variable once (at the top), then Set / Increment / Append to it as the flow runs — a running total, a counter, or a built-up HTML string.

Initialize variable → name: TotalRevenue, type: Float, value: 0 (inside Apply to each) Increment variable → TotalRevenue by items('Apply_to_each')?['Revenue']
▸Scope & parallel branch Group and split Logic

Scope groups several actions into one block — useful for a single "try" you can wrap error handling around (set the next Scope to run only if the previous failed). A parallel branch runs two paths at the same time.

▸Expressions Functions for the tricky bits Advanced

When dynamic content isn't enough, write an expression in the formula box. Common ones:

utcNow() // current timestamp formatDateTime(utcNow(),'dd/MM/yyyy') addDays(utcNow(), 7) // a week from now concat('Order ', triggerBody()?['Subject']) if(equals(item()?['Priority'],'High'), 'Escalate', 'Log') length(body('List_rows')?['value']) // how many rows came back
▸Approvals Get a human yes/no Action

The Start and wait for an approval action pauses the flow, emails an approver, and continues based on their response — perfect for sign-off on an escalation or a change window.

Design challenges

Build the flow, then check the key

For each scenario, sketch the flow on paper or in the designer — which flow type, which trigger, which actions, in what order. Then reveal the worked answer.

Challenge 1 — Notify & log new orders

Whenever an email arrives in the shared Delivery mailbox with "GOLD order" in the subject, post a heads-up in the Delivery Teams channel and add a row to the order-tracking spreadsheet.

Hint: which flow type starts "whenever an email arrives"?

Show answer flow

Automated cloud flow.

TRIGGER · Outlook When a new email arrives (V3) — folder: Delivery; Subject Filter: GOLD order
ACTION · Teams Post message in a chat or channel → channel "Delivery", message uses dynamic From and Subject
ACTION · Excel Online Add a row into a table → map Subject, Received time, From

The subject filter on the trigger means the flow never fires on unrelated mail — cheaper and cleaner than a Condition after the fact.

Challenge 2 — Weekly "due this week" digest

Every Monday at 08:00, email yourself a list of orders whose Target Go-Live falls in the next 7 days, pulled from an Excel table.

Hint: a clock-based start, and the data comes back as a list.

Show answer flow

Scheduled cloud flow.

TRIGGER · Schedule Recurrence → Weekly, Monday, 08:00
ACTION · Excel Online List rows present in a table → with a Filter Query on Target Go-Live within the next 7 days
ACTION · Data ops Create HTML table from the returned rows
ACTION · Outlook Send an email (V2) → to yourself, body = the HTML table

Filtering in the List rows step (rather than looping and checking each) keeps it fast. Create HTML table turns the array into a tidy email without an Apply to each.

Challenge 3 — Escalate only the High-priority holds

When an order in a SharePoint list is changed to status On Hold: if its Priority is High, request approval from the Team Manager to escalate; otherwise just add a comment/log entry. Nothing should happen for other status changes.

Hint: you'll need a trigger filter and a branch.

Show answer flow

Automated cloud flow with a trigger condition + a Condition control.

TRIGGER · SharePoint When an item is created or modified → add a trigger condition so it only continues when Status = "On Hold"
CONDITION Priority is equal to High
If YES → Start and wait for an approval (approver: Team Manager) then Update item / Send an email with the decision If NO → Add a comment / append a row to the log table

The trigger condition stops the flow from running on every edit; the Condition then routes by priority. Tip: also compare to the item's previous Status if you only want the moment it becomes On Hold.

Challenge 4 — A legacy portal with no connector

You need to open the GOLD order portal in a browser, search an order number, copy its status, and paste it into an Excel sheet. There is no GOLD connector or API. How do you automate it?

Hint: this one isn't a cloud flow.

Show answer flow

Desktop flow (RPA) in Power Automate Desktop — it automates the screen itself.

UI Launch / attach to the browser → navigate to the GOLD URL
UI Populate text field with the order number → Press button (Search)
UI Extract data / Get details of element → read the status text into a variable
Excel Launch Excel → Write to cell → Save & close

RPA is the right tool only because there's no API. It's more brittle than a cloud flow (a page redesign can break it), so add waits for elements to load and handle the "not found" case.
