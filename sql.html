Clauses by level

Build your SQL vocabulary

SQL reads and summarises data in relational tables — the model from the Foundations page. This track is split into three levels — tap any card to switch. Start at Level 1 and climb. Each clause has a worked example on the orders table and a task to try yourself — paste your answer into the runner below to check it.

▸SELECT / FROM Choose columns Basic

Pick the columns you want from a table. * means "all columns"; otherwise list names separated by commas.

SELECT customer, region, revenue FROM orders;

Try it

Return just the site_code and city for every order.

Show answer
SELECT site_code, city FROM orders;

Column names go after SELECT; the table after FROM.

▸WHERE Filter rows Basic

Keep only the rows that match a condition. Text values go in 'single quotes'.

SELECT customer, revenue FROM orders WHERE region = 'EMEA' AND status = 'Delivered';
i
Operators: = equals, <> not equal, > < >= <=, and AND / OR to combine conditions.

Try it

List the customer and revenue of every order in the APAC region.

Show answer
SELECT customer, revenue FROM orders WHERE region = 'APAC';
▸ORDER BY / LIMIT Sort and cap Basic

Sort by a column (DESC for high-to-low, ASC for low-to-high) and keep just the top few with LIMIT.

SELECT customer, site_code, revenue FROM orders ORDER BY revenue DESC LIMIT 5;

Try it

Show the 3 smallest orders by revenue (customer and revenue).

Show answer
SELECT customer, revenue FROM orders ORDER BY revenue ASC LIMIT 3;

ASC is the default sort, but spelling it out makes intent clear.

▸DISTINCT Unique values Basic

Remove duplicates so each value appears once — handy for "what values exist in this column?".

SELECT DISTINCT country FROM orders ORDER BY country;

Try it

List each service_type exactly once.

Show answer
SELECT DISTINCT service_type FROM orders;
▸Aggregates + GROUP BY Summarise Intermediate

COUNT, SUM, AVG, MIN and MAX collapse many rows into one. GROUP BY does it per category, and AS renames the output column.

SELECT region, COUNT(*) AS orders, ROUND(SUM(revenue)) AS total_rev, ROUND(AVG(fte_days),1) AS avg_fte FROM orders GROUP BY region ORDER BY total_rev DESC;

Try it

For each service_type, show the number of orders and total revenue, busiest first.

Show answer
SELECT service_type, COUNT(*) AS orders, ROUND(SUM(revenue)) AS total_rev FROM orders GROUP BY service_type ORDER BY orders DESC;
▸HAVING Filter groups Intermediate

WHERE filters rows before grouping; HAVING filters the groups after.

SELECT dpm, COUNT(*) AS orders FROM orders GROUP BY dpm HAVING COUNT(*) >= 10 ORDER BY orders DESC;
i
Order of execution: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT. That's why a SELECT alias can't be used in WHERE — it doesn't exist yet.

Try it

Which customers have more than 8 orders? Show the customer and the count.

Show answer
SELECT customer, COUNT(*) AS orders FROM orders GROUP BY customer HAVING COUNT(*) > 8 ORDER BY orders DESC;
▸JOIN Combine tables Intermediate

Match rows from two tables on a shared key. Here we compare each region's actual revenue to its target from the targets table.

SELECT o.region, ROUND(SUM(o.revenue)) AS actual, t.revenue_target, ROUND(100.0 * SUM(o.revenue) / t.revenue_target) AS pct_of_target FROM orders o JOIN targets t ON o.region = t.region GROUP BY o.region ORDER BY actual DESC;

Try it

Show each region with its actual total revenue and its revenue_target side by side.

Show answer
SELECT o.region, ROUND(SUM(o.revenue)) AS actual, t.revenue_target FROM orders o JOIN targets t ON o.region = t.region GROUP BY o.region;

Table aliases (o, t) keep the column references short and unambiguous.

▸CASE If-then logic in a column Advanced

Bucket rows into categories on the fly, then group by the result.

SELECT CASE WHEN revenue >= 30000 THEN 'large' WHEN revenue >= 15000 THEN 'mid' ELSE 'small' END AS band, COUNT(*) AS orders FROM orders GROUP BY band ORDER BY orders DESC;

Try it

Label each order 'High' when priority is 'High', otherwise 'Other', and count each label.

Show answer
SELECT CASE WHEN priority = 'High' THEN 'High' ELSE 'Other' END AS bucket, COUNT(*) AS n FROM orders GROUP BY bucket;
▸Subquery A query inside a query Advanced

Use the result of one query inside another. Here: orders above the overall average revenue.

SELECT customer, site_code, revenue FROM orders WHERE revenue > (SELECT AVG(revenue) FROM orders) ORDER BY revenue DESC;

Try it

List orders whose revenue is above the average revenue of EMEA orders.

Show answer
SELECT customer, revenue FROM orders WHERE revenue > (SELECT AVG(revenue) FROM orders WHERE region = 'EMEA') ORDER BY revenue DESC;
▸CTE (WITH) Name a step for readability Advanced

A common table expression is a named temporary result you can build on — cleaner than nesting subqueries.

WITH region_rev AS ( SELECT region, SUM(revenue) AS total FROM orders GROUP BY region ) SELECT region, total FROM region_rev ORDER BY total DESC;

Try it

Using a CTE of orders-per-DPM, return only the DPMs with 10 or more orders.

Show answer
WITH c AS ( SELECT dpm, COUNT(*) AS n FROM orders GROUP BY dpm ) SELECT * FROM c WHERE n >= 10 ORDER BY n DESC;

Because the CTE already computed n, you can filter on it with a plain WHERE.

▸Window functions Rank without collapsing rows Advanced

Aggregate across rows while keeping every row. RANK() OVER (...) ranks each order by revenue within its region.

SELECT region, customer, revenue, RANK() OVER (PARTITION BY region ORDER BY revenue DESC) AS rev_rank FROM orders ORDER BY region, rev_rank LIMIT 15;

Try it

Rank all orders by revenue (highest = 1), showing customer, revenue and the rank.

Show answer
SELECT customer, revenue, RANK() OVER (ORDER BY revenue DESC) AS rnk FROM orders ORDER BY rnk LIMIT 15;

Drop PARTITION BY to rank across the whole table instead of within each group.

Good habits

A few things that trip people up

Single vs double quotes — text values use 'single quotes'. Double quotes are for identifiers (column/table names), not strings.
= vs LIKE — = is exact. For partial matches use LIKE 'SD-WAN%' (% = any characters, _ = one character).
NULL is special — use IS NULL / IS NOT NULL, never = NULL. Here, actual_go_live is NULL for orders not yet delivered.
Integer division — 3/2 can return 1. Multiply by 1.0 (e.g. 100.0 * a / b) to force decimals.
COUNT(*) vs COUNT(col) — COUNT(*) counts rows; COUNT(col) ignores NULLs in that column.
