Functions by level

Build your formula vocabulary

This track is split into three levels — tap any card to switch. Start at Level 1 and climb. Each function has a worked example on the sample data and a task to try yourself before revealing the answer.

▸SUM / AVERAGE / COUNT The starter pack Basic

SUM adds numbers, AVERAGE means them, COUNT counts cells that contain numbers, and COUNTA counts cells that aren't empty (text included).

=SUM(O2:O73) // total revenue =AVERAGE(N2:N73) // average FTE days =COUNTA(A2:A73) // how many orders (counts text Order IDs)

Use COUNT when you're counting a number column, COUNTA when you're counting rows by a text key like Order ID.

Try it

How much total Revenue (EUR) is in the dataset, and how many orders are there in total?

Show answer
=SUM(O2:O73) =COUNTA(A2:A73)

Revenue lives in column O, Order IDs in column A. COUNTA beats COUNT here because Order IDs are text (10-1735…).

▸MIN / MAX Find the extremes Basic

Return the smallest and largest number in a range. Handy for "biggest order" or "earliest date".

=MAX(O2:O73) // largest order value =MIN(K2:K73) // earliest Order Date

Try it

What is the largest single order value, and the highest number of APs migrated in any one order?

Show answer
=MAX(O2:O73) // Revenue column =MAX(P2:P73) // APs Migrated column
▸IF Make a decision Basic

The logic workhorse: =IF(test, value_if_true, value_if_false).

=IF(H2="Delivered","Closed","Open") =IF(O2>20000,"High value","Standard")
i
Comparison operators: = equals, <> not equal, >= <= at least / at most.

Try it

Add a column that flags an order as "Late" if Actual Go-Live (M) is after Target Go-Live (L), otherwise "On time". Watch out for blank Actual dates.

Show answer
=IF(M2="","Not delivered yet",IF(M2>L2,"Late","On time"))

A blank Actual Go-Live means the order isn't delivered, so we test for "" first to avoid mislabelling it.

▸& / TEXTJOIN Stick text together Basic

The & operator glues text. TEXTJOIN does the same with a delimiter and can ignore blanks.

=B2&" — "&D2 // "SKF Industrial — Gothenburg" =TEXTJOIN(" / ",TRUE,C2,E2,G2) // "SEGOT / Sweden / LAN Refresh"

Try it

Build a one-line label for each order: Customer (Site Code) – Service Type.

Show answer
=B2&" ("&C2&") – "&G2
▸$ absolute references Lock a cell Basic

When you copy a formula down, references shift. A dollar sign locks a row or column so it doesn't move. $O$2 locks both; O$2 locks the row only; $O2 locks the column only.

=O2/$O$74 // each order's share of the grand total in O74
⌨
Press F4 while editing a reference to cycle through the lock options.

Try it

Put the grand total revenue in O74, then add a column showing each order's percentage of that total, formatted as a %.

Show answer
O74: =SUM(O2:O73) R2 : =O2/$O$74 // then format column R as Percentage, fill down

The $O$74 stays put as you fill down, while O2 walks to O3, O4…

▸ROUND Tidy the decimals Basic

=ROUND(number, digits) — positive digits round after the decimal, 0 rounds to a whole number.

=ROUND(N2,0) // FTE days to a whole number =ROUND(AVERAGE(N2:N73),1) // average FTE, one decimal
!
Rounding for display is better done with cell formatting — ROUND actually changes the value, which matters if you sum the results.
▸COUNTIF / SUMIF Count & total with a condition Intermediate

Add one criterion to a count or a sum.

=COUNTIF(H2:H73,"Delivered") // how many delivered orders =SUMIF(G2:G73,"LAN Refresh",O2:O73) // total revenue from LAN Refresh

Pattern for SUMIF: =SUMIF(range_to_test, criteria, range_to_add).

Try it

How many orders are On Hold, and what is the total revenue for the EMEA region?

Show answer
=COUNTIF(H2:H73,"On Hold") =SUMIF(F2:F73,"EMEA",O2:O73)
▸COUNTIFS / SUMIFS / AVERAGEIFS Many conditions Intermediate

The -IFS versions take multiple criteria pairs. Note the argument order flips: in SUMIFS the sum range comes first.

=SUMIFS(O2:O73, G2:G73,"SD-WAN Migration", F2:F73,"APAC") =COUNTIFS(H2:H73,"Delivered", I2:I73,"Ahmed El Bourgy") =AVERAGEIFS(N2:N73, G2:G73,"LAN Refresh")
i
Criteria can use wildcards and operators in quotes: ">20000", "<>Cancelled", "SE*" (starts with SE).

Try it

What is the total revenue of High priority orders that are not Cancelled?

Show answer
=SUMIFS(O2:O73, Q2:Q73,"High", H2:H73,"<>Cancelled")

Priority is column Q, Status is column H. "<>Cancelled" means "not equal to Cancelled".

▸XLOOKUP The modern lookup Intermediate

XLOOKUP replaces VLOOKUP. It looks up a value and returns a matching value from another column — in any direction, with a clean "if not found" option.

=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found]) =XLOOKUP("10-1735122", A2:A73, B2:B73, "Not found") // returns the Customer for that Order ID

It works right-to-left too, unlike VLOOKUP, because you point at the lookup and return columns separately.

Try it

Given an Order ID typed in cell T1, return its Status and its DPM.

Show answer
=XLOOKUP(T1, A2:A73, H2:H73, "No such order") // Status =XLOOKUP(T1, A2:A73, I2:I73, "No such order") // DPM
▸VLOOKUP & INDEX/MATCH The classics Intermediate

You'll still meet these in older files. VLOOKUP searches the first column of a block and returns a column to its right:

=VLOOKUP("10-1735122", A2:Q73, 8, FALSE) // 8 = the 8th column of the block (Status). FALSE = exact match.

INDEX/MATCH is more flexible — it can look in any direction and survives inserted columns:

=INDEX(H2:H73, MATCH("10-1735122", A2:A73, 0)) // MATCH finds the row, INDEX returns the Status from it. 0 = exact.
!
VLOOKUP's column number breaks if someone inserts a column. Prefer XLOOKUP or INDEX/MATCH in anything you'll maintain.
▸IFERROR Catch the #N/A Intermediate

Wrap any formula to replace errors with something friendly.

=IFERROR(XLOOKUP(T1,A2:A73,B2:B73), "Check the Order ID")

With XLOOKUP you'd usually use its built-in 4th argument instead; IFERROR shines around division and older lookups.

▸TEXT & date functions Format and slice dates Intermediate

Dates are just numbers under the hood. Pull pieces out, or do date maths.

=YEAR(K2) =MONTH(K2) =DAY(K2) =M2-L2 // days late/early (Actual minus Target) =TEXT(K2,"mmm yyyy") // "Mar 2025" — for grouping/labels =NETWORKDAYS(K2,L2) // working days between order and target

Try it

For each delivered order, how many days early or late was it? Then label the month of each Order Date as e.g. "Jan 2025".

Show answer
=IF(M2="","",M2-L2) // negative = early, positive = late =TEXT(K2,"mmm yyyy")
▸PivotTables Summarise without formulas Intermediate

A PivotTable groups and totals data by drag-and-drop — no formulas needed. With the data selected: Insert ▸ PivotTable.

Drop zoneWhat to put thereResult
RowsRegion, then Service TypeOne block per region, broken down by service
ValuesRevenue (EUR) → SumTotals in the grid

Filters / SlicersStatusToggle Delivered vs all
i
Right-click a value ▸ Show Values As ▸ % of Grand Total turns raw totals into shares. Add a Slicer (PivotTable Analyze ▸ Insert Slicer) for clickable filters.

Try it

Build a PivotTable showing total revenue and order count by Customer, sorted highest revenue first, filtered to Delivered orders only.

Show answer

Rows = Customer. Values = Revenue (Sum) and Order ID (Count). Add Status as a slicer or filter and pick Delivered. Click the revenue total's dropdown ▸ Sort Largest to Smallest.

▸Conditional formatting Let the sheet flag things Intermediate

Home ▸ Conditional Formatting. Colour cells by rule, add data bars or icon sets, or write a formula rule.

// Highlight late deliveries — "Use a formula" rule, applied to the row: =AND($M2<>"", $M2>$L2)

Lock the column with $ (so the whole row highlights), but leave the row number free so the rule walks down.

▸FILTER Return whole rows by condition Advanced

A dynamic array function: returns every row that matches, spilling automatically. No dragging.

=FILTER(A2:Q73, (F2:F73="EMEA")*(H2:H73="Delivered"), "No matches") // multiply conditions for AND, add (+) them for OR

Try it

Spill a list of every High-priority order that is still In Progress or Staging.

Show answer
=FILTER(A2:Q73, (Q2:Q73="High")*((H2:H73="In Progress")+(H2:H73="Staging")), "None")

* is AND, + is OR. Wrap the OR group in its own parentheses.

▸UNIQUE / SORT / SORTBY Lists that maintain themselves Advanced
=SORT(UNIQUE(B2:B73)) // distinct customers, A–Z =SORTBY(A2:A73, O2:O73, -1) // Order IDs sorted by revenue, high→low

Combine them: a clean dropdown source, a top-N list, a deduped lookup table — all live.

Try it

Produce a sorted, duplicate-free list of every Country in the data.

Show answer
=SORT(UNIQUE(E2:E73))
▸LET Name your steps Advanced

LET assigns names to intermediate results — formulas become readable and faster (each value is computed once).

=LET( rev, O2:O73, status,H2:H73, live, FILTER(rev, status="Delivered"), AVERAGE(live) ) // average revenue of delivered orders, written legibly
i
Pattern: pairs of name, value, then a final expression that uses them.
▸LAMBDA Build your own function Advanced

LAMBDA turns a formula into a reusable function. Define it once in Name Manager (Formulas ▸ Define Name), then call it like any built-in.

// Name: DaysLate Refers to: =LAMBDA(target, actual, IF(actual="","",actual-target)) // then in the sheet: =DaysLate(L2, M2)

Pair with MAP, BYROW or REDUCE to apply a LAMBDA across a whole array at once.

▸SUMPRODUCT Weighted & multi-condition maths Advanced

Multiplies arrays element-by-element then sums — great for weighted averages and conditions in older Excel.

=SUMPRODUCT((F2:F73="EMEA")*(H2:H73="Delivered")*O2:O73) // total delivered EMEA revenue, no SUMIFS needed =SUMPRODUCT(N2:N73, O2:O73)/SUM(N2:N73) // revenue weighted by FTE days

Try it

Use SUMPRODUCT to count how many orders are both Wireless Migration and have more than 50 APs migrated.

Show answer
=SUMPRODUCT((G2:G73="Wireless Migration")*(P2:P73>50))
▸Tables & structured references Stop hard-coding ranges Advanced

Convert your range to a Table (Ctrl+T), name it Projects, and formulas read like English — and auto-expand when rows are added.

=SUM(Projects[Revenue (EUR)]) =SUMIFS(Projects[Revenue (EUR)], Projects[Region],"EMEA") =COUNTIFS(Projects[Status],"Delivered")
✓
This is the single biggest upgrade to a messy workbook: no more O2:O73 that silently misses new rows.
Tips & tricks

Small moves, big time savings

Flash Fill Ctrl+E

Type the result you want in one cell (e.g. an initials code), start the next, and Excel detects the pattern and fills the rest. Perfect for splitting or reformatting text without formulas.

Tables Ctrl+T

Turns a range into a self-expanding, filterable, named object. Formulas referencing it auto-grow with new rows.

Freeze panes

View ▸ Freeze Panes ▸ Freeze Top Row so headers stay visible while you scroll 72 rows of orders.

Paste Special Ctrl+Alt+V

Paste Values to drop formulas, or Transpose to flip rows and columns.

Quick fill down Ctrl+D

Select a cell plus the range below it and fill the formula down instantly. Ctrl+R fills right.

Jump & select

Ctrl+↓ jumps to the bottom of a column; add Shift to select on the way. Ctrl+; stamps today's date.

Data from the web

Pull live data into a sheet

Excel can read tables straight off a web page or an online file and refresh them on demand — no copy-paste.

From a web page

Data ▸ Get Data ▸ From Other Sources ▸ From Web. Paste the URL, and the Navigator shows any tables Excel finds. Pick one, Transform to clean it, then Load.

It lands as a query you can refresh with Data ▸ Refresh All (Ctrl+Alt+F5).

From a file or folder

Data ▸ Get Data ▸ From File reads a CSV, another workbook, or an entire folder of files — combining them into one table automatically. Great for "every weekly export in this folder".

i
Anything you import this way becomes a Power Query — which is exactly the next section.
Power Query

Clean once, refresh forever

Power Query (Data ▸ Get & Transform) records every cleaning step you take. Next month, point it at the new file and click Refresh — all your steps replay automatically. This is the antidote to repetitive manual cleanup.

The mental model

You're not editing data — you're recording a recipe. Each action becomes a step in the Applied Steps list on the right. The steps run top to bottom every refresh, so the source can change but your cleaning stays.

Common steps you'll use constantly

GoalWhereWhat it does
Promote headersHome / TransformUse the first row as column names
Change typeTransform ▸ Data TypeSet Date, Whole Number, Text — fixes text-dates and EU decimals
Trim / Clean / LowercaseTransform ▸ FormatStrip stray spaces and unify casing ("EMEA" vs " emea ")
Remove duplicatesHome ▸ Remove RowsDrop repeated rows by selected columns
Replace / Fill downTransformPatch blanks; carry a value down into empty cells
Split / Merge columnsTransformBreak "City, Country" apart or join codes together
UnpivotTransform ▸ UnpivotTurn wide month-columns into tidy rows — essential for charts
Group ByTransform ▸ Group BySummarise (sum revenue per region) right inside the query

i
Behind the steps is the M language (View ▸ Advanced Editor). You rarely write it by hand at first, but it's worth peeking to see what each click generated.

Try it — clean the messy workbook

Download the Messy workbook at the top of this page and open a blank file ▸ Data ▸ Get Data ▸ From File ▸ From Excel Workbook ▸ pick it ▸ select the DATA sheet ▸ Transform Data. (Or use Data ▸ From Text/CSV with the messy CSV.) The START HERE sheet lists what's wrong. Fix every issue in the query, then Close & Load.

Show the answer key (step order)
  1. Use First Row as Headers (Transform) — turn the top row into column names.
  2. Order Date ▸ Data Type ▸ Date (use Using Locale → English (UK) so dd/mm/yyyy parses correctly).
  3. FTE Days ▸ Replace Values "," → "." first, then Data Type ▸ Decimal Number.
  4. Customer ▸ Transform ▸ Format ▸ Trim, then Capitalize Each Word (kills " harman international " → "Harman International").
  5. Status ▸ Format ▸ lowercase then Capitalize Each Word (unifies "DELIVERED", "in-progress", "On Hold").
  6. Remove Duplicates — select all columns ▸ Home ▸ Remove Rows ▸ Remove Duplicates. The messy file has exact-duplicate rows; this is the step that drops them. (Tip: to dedupe by a key like Order ID instead, select just that column first.)
  7. Country ▸ Transform ▸ Fill ▸ Down (or Replace blanks) to patch empty cells.
  8. Home ▸ Close & Load.

Why this matters: next month, drop in the new export and click Refresh — all eight steps replay, including the de-duplication. Zero manual cleanup.

PivotTables

PivotTables — OLAP on your desktop

A PivotTable is the fastest way to turn a flat sheet into answers — drag a field to summarise, with no formulas at all. It's also the clearest hands-on example of OLAP: you're slicing a measure (revenue, count) by dimensions (region, service, month). If the cube idea is new, read the OLAP explanation in Foundations first — this section is that theory made clickable.

The four drop zones — and their OLAP meaning

Rows & Columns

The categories you group by — Region, Service Type, Customer, Month. In OLAP these are the dimensions: the edges of the cube.

Values

The number in each cell — Sum of Revenue, Count of Orders, Average FTE. This is the OLAP measure (an aggregation).

Filters & Slicers

Restrict to part of the data — only Delivered, only EMEA. Fixing one dimension is an OLAP slice; several at once is a dice.

Expand / collapse

Open a Region to see its Sites, or collapse back to totals. That's OLAP drill-down and roll-up.

Build one in six steps

  1. Click any cell in the data, then Insert ▸ PivotTable ▸ OK — a new sheet appears with the field list.
  2. Drag Region into Rows.
  3. Drag Service Type into Columns.
  4. Drag Revenue (EUR) into Values — then click it ▸ Value Field Settings ▸ set to Sum (Excel guesses Count if the column has any blanks).
  5. Drag Status into Filters, or add a Slicer (PivotTable Analyze ▸ Insert Slicer) for clickable filtering.
  6. Right-click any value ▸ Show Values As ▸ % of Grand Total to turn euros into shares.

PivotTable ⇄ OLAP, side by side

★
Every PivotTable answers an OLAP question — "a measure by a dimension by a dimension" (revenue by region by quarter). See the worked OLAP questions in Foundations and notice each one is simply a PivotTable layout.
!
Refresh, don't rebuild: a PivotTable is a snapshot. When the source rows change, right-click ▸ Refresh. Build it on a Table (Ctrl+T) so rows added below are picked up automatically.

Try it — an OLAP question as a PivotTable

Build a PivotTable of total Revenue by Customer (Customer in Rows, Sum of Revenue in Values), filtered to Delivered orders, sorted highest first. Which customer comes out on top?

Show answer

Customer → Rows; Revenue (EUR) → Values (Sum); Status → Filters = Delivered; then open the Customer dropdown ▸ Sort Largest to Smallest by Sum of Revenue. The top customer is Carlsberg Group. In OLAP terms, you sliced the cube to Delivered and ranked the Revenue measure along the Customer dimension.

Analysis drills — work them, then reveal the key

Drill 1 · How many orders are On Hold right now?

Show the key

Reach for COUNTIF — count cells in one column that match one condition.

=COUNTIF(H2:H73,"On Hold")

Drill 2 · Total Revenue (EUR) of all Delivered orders.

Show the key

SUMIF — sum one column where another column matches a condition. (Criteria range first, then criteria, then the range to sum.)

=SUMIF(H2:H73,"Delivered",O2:O73)

Drill 3 · Total revenue of High-priority orders that are not Cancelled.

Show the key

Two conditions → SUMIFS. Note the "<>Cancelled" operator for "not equal to".

=SUMIFS(O2:O73, Q2:Q73,"High", H2:H73,"<>Cancelled")

Drill 4 · Return the Status of order 10-1735122 — and don't let it break if someone inserts a column.

Show the key

Use XLOOKUP (or INDEX/MATCH). Both point at columns directly, so an inserted column won't shift the result. Avoid VLOOKUP with a hard-coded column number here — that's exactly what breaks.

=XLOOKUP("10-1735122", A2:A73, H2:H73)

Drill 5 · Look up an order's Customer by Order ID, with a friendly message if the ID doesn't exist.

Show the key

XLOOKUP has a built-in "if not found" argument — no need to wrap it in IFERROR (which would also hide real errors).

=XLOOKUP(T1, A2:A73, B2:B73, "No order with that ID")

Drill 6 · Flag every row as Late / On time / Not delivered yet (Actual vs Target Go-Live).

Show the key

A nested IF: first test whether it's delivered at all, then compare the two dates.

=IF(M2="","Not delivered yet", IF(M2>L2,"Late","On time"))

Drill 7 · A self-updating, duplicate-free, A–Z list of every Customer.

Show the key

UNIQUE strips duplicates, SORT orders them — and because they spill, the list rebuilds itself when the data changes. No "Remove Duplicates" button to re-run.

=SORT(UNIQUE(B2:B73))

Drill 8 · Spill a table of only EMEA + Delivered orders — Order ID, Customer, Revenue.

Show the key

FILTER with two conditions multiplied together (* = AND). Use HSTACK to choose just the three columns you want.

=FILTER(HSTACK(A2:A73,B2:B73,O2:O73), (F2:F73="EMEA")*(H2:H73="Delivered"), "None")

No HSTACK? Use CHOOSE({1,2,3},A2:A73,B2:B73,O2:O73) in its place.

Drill 9 · Total revenue of Delivered EMEA orders — in one formula, no helper column.

Show the key

SUMIFS handles it directly. (SUMPRODUCT is the old-school alternative: =SUMPRODUCT((H2:H73="Delivered")*(F2:F73="EMEA")*O2:O73).)

=SUMIFS(O2:O73, H2:H73,"Delivered", F2:F73,"EMEA")

Drill 10 · A PivotTable: Revenue by Region (rows) × Service Type (columns).

Show the key

Not a formula — a tool. Click any cell in the data ▸ Insert ▸ PivotTable. Then drag fields: Region → Rows, Service Type → Columns, Revenue (EUR) → Values (set it to Sum, not Count). Right-click a value ▸ Number Format for clean euros.
